Form Submission

Submit a form with contact details and optional file attachments. Requires a valid API key.

POST /api/submit

Authentication — Pass the API key via:

  • Header: X-API-Key: your_api_key
  • Header: Authorization: Bearer your_api_key
  • Query: ?api_key=your_api_key
  • Body (JSON/form): api_key: "your_api_key"

Form fields

FieldTypeDescription
formPurposestringPurpose of the form
firstNamestringFirst name
lastNamestringLast name
emailstringEmail address
phonestringPhone number
helpNeededOfferedstringHelp needed or offered
additionalMaterialsfile[]Optional file uploads (max 10 files, 10MB each)

Request (multipart/form-data)

POST /api/submit
Content-Type: multipart/form-data
X-API-Key: your_api_key

formPurpose=Tech support
firstName=John
lastName=Doe
email=john@example.com
phone=555-1234
helpNeededOffered=Need help with email
additionalMaterials=<file>

Request (JSON, no files)

POST /api/submit
Content-Type: application/json
X-API-Key: your_api_key

{
  "formPurpose": "Tech support",
  "firstName": "John",
  "lastName": "Doe",
  "email": "john@example.com",
  "phone": "555-1234",
  "helpNeededOffered": "Need help with email"
}

  "message": "Form submitted successfully"

{
  "error": "Invalid or missing API key"
}

Admin Page

Admin dashboard showing all form submissions. Requires admin key as query parameter.

GET /admin?key=YOUR_ADMIN_KEY

Access with your admin key (stored in admin_pass.txt or ADMIN_KEY env var). Env vars use UPPERCASE: API_KEY, ADMIN_KEY.

Submissions API (Admin)

Returns all submissions as JSON. Admin only.

GET /api/submissions?key=YOUR_ADMIN_KEY

Response: Array of submission objects.
